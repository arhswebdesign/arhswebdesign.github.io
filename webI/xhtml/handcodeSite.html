Website Due Date: Friday, 5/11/18

Creating the web site:

Remember our discussions of good design.

  • Use the Web Design Criteria Handout handout as a guide for building the site.
  • Site should demonstrate "don't make me think" qualities
  • The content, language and images, must be school appropriate.
  • Create a simple, but effective file naming structure for the pages and images used.

Choose a structure for the site:

  • How many pages will you need?
  • What kind of structure will work best for this site?
  • Do you want your visitors to go through the site in a particular direction, or do you want to make it easy for them to explore in any direction?

General requirements for the site:

  • Do Not Build any part of the site before complete plans are submitted. Doing so will cost the final grade of the project 10% right off the top.
  • Make a fully styled & designed template with external style sheet before building your first page. This saves typing, because you can start with a copy of this template for each of the pages in the site
  • Organize all files for this site into its own folder. In other words, create a folder for the website and organize all files for the site in that folder.
  • The site should be completely portable. In other words, we should be able to copy the folder, upload onto a computer not on the school network and everything in the site still functions.
  • Create an images folder in the website folder and put all images in it.
  • The file name of the first page of a web site should be named index.html. This is a general rule which is true for most web hosts.This file must be in the "root" folder, the main folder that holds the web site Do not put it in deeper in a folder within the root folder.
  • Use tables to create the layout.
  • Give each page a unique title, in the <title> tag.
  • Use external style sheets to style all pages in the site.
  • Be sure all links work.
  • Include alternate text for all images.
  • Include an email link.
  • Navigation should be easily accessible and simple to manage.
  • Organize your html code so that it is easy to read.
  • Include html code comments and CSS code comments to explain what each section is about.
