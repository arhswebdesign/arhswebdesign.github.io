Introduction to Working in Web Design

To begin

  • Press Windows-E, open ThisPC and then open S:\HS\Student\WebDesign
  • Pause to study the path to that folder. Each name, separated by a slash, is a folder. It's a series of folders within folders. We will be using and thinking about pathnames frequently in this class.
  • Look for a folder inside S:\HS\Student\WebDesign with your last name.
  • You will save all of your web design work in this portfolio/folder.
  • Create other folders inside your web design portfolio. Name these as follows: early_work, middle_work, later_work, first_site, final_site, other.
  • early_work can hold the files from the first assignments up to intro. styling
  • middle_work can contain the next few assignments up to layout table.
  • later_work can contain the next assignments up to the "cascading style sheets" assignment.
  • Any assignment where you have more than one file, make a separate folder for the assignment.

The shared drive honor system, S:\

  • Work in your folder only
  • Good idea to back up your work to your H:\ once a week. Simply copy your folder from the shared drive and paste it into your H:\.
  • Report any problems with S:\, asap.
  • The computer network is monitored all day, every day. When there is a problem with files deleted from a shared drive, we can find out who deleted the files within minutes.

Daily routines

  • Log into your computer as bell rings
  • Open Brackets, the class web site, the W3Schools web site, and your folder on S:\HS\Student\webDesign.
  • Keep these open throughout the class. Add the W3Schools web site to your favorites so that it is easily accessible.
  • Turn off monitors during teacher talk.
  • As the trimester progresses, there is less and less teacher talk and more hands on work.
  • Working rhythm: write html code in a Brackets file, save it as .html file, and use the lightning bolt to open and view that page in a web browser. Write code, view. Write code, view.
  • This allows you to see changes to the web page you are building.

Naming your files:

  • When you save a web document, whether it is a page, an image, or an object, give it a name related to the assignment, a name that makes sense, in the correct format.
  • Name your assignments with short, descriptive file names.
  • For example, the second file created in assignment 1 should be named something like basic_tags.html
  • Use the underscore character instead of spaces between words in a file name.

Using W3 Schools online tutorials for learning and practicing html and css

  • Study the examples that site shows us and then practice writing the code.
  • Do not simply copy and paste information from W3Schools into your .html files. You won't learn anything by doing this and it is very obvious when someone has copy/pasted.
  • Since copy/pasted work is not yours, it is plagiarized, and will result in a 0 for that assignment. All school rules regarding plagiarism will be followed in all cases of plagiarism.

Using the class web site effectively

  • Browse around the site to learn where things are stored.
  • Read each assignment completely and pay close attention to the details.
  • Make sure each assignment is completed before moving on to the next assignment. Use the assigned tasks as a checklist to keep track of your work.
  • Add the site to your favorites.

use the help list on the white board

  • Asking for help instead of thinking for yourself and before trying to solve problems is a sign of laziness
  • Use the help board when you need help but do work on your troubleshooting skills by trying to solve the problem yourself.
